# ----------------------------------------
#  Type 종류
# ----------------------------------------
# ADD       : 새로운 코드나 파일 추가
# MODIFY    : 기존 기능 수정
# FIX       : 버그 수정
# DOCS      : 문서(주석, README 등) 수정
# REMOVE    : 코드나 파일 삭제
# IMPLEMENT : 무언가의 전체 동작을 "구현"했을 때 사용 (ADD보다 큰 단위)
# TEST      : 테스트 코드 추가 및 수정
# RENAME    : 파일명, 변수명, 함수명 등의 이름 변경
# MOVE      : 코드나 파일 이동
# CHORE     : 빌드/패키지 매니저 설정 등 기타 작업

# ----------------------------------------
#  제목(Subject) 작성 규칙
# ----------------------------------------
# - 제목 작성 후 반드시 한 줄 띄워서 본문과 구분
# - 제목은 대문자로 시작, 마침표는 쓰지 않음
# - 제목은 명령문으로 작성

# ----------------------------------------
#  본문(Body) 작성 규칙
# ----------------------------------------
# - 본문 작성 후 한 줄 띄워서 Resolves 영역과 구분
# - 각 행은 72자 이내로 작성
# - "무엇을", "왜" 했는지를 중심으로 작성
# - 본문은 여러 줄로 구성 가능
